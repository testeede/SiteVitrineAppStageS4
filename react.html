Version 1.0 - React

Première version développée pour remplacer le processus manuel de validation

Objectifs de la Version 1.0

L'objectif de cette version était de remplacer le processus manuel de validation des alertes de fraude par un outil web permettant de valider les alertes de fraude de manière plus efficace et rapide.

L'application était initialement développée en React avec intégration à l'application existante "Memorize".

Caractéristiques Techniques

Framework

Développée en React avec intégration à l'application existante "Memorize"

Limitations

Bugs et lenteurs importantes, communication indirecte avec le NAS

Fichiers

Fonctionnait uniquement avec des fichiers locaux via l'API File System Access

Problèmes Identifiés

  • ❌ Lenteurs importantes lors de l'accès aux fichiers
  • ❌ Forte dépendance au code existant
  • ❌ Interface quasi-inutilisable sur les dossiers non-locaux à cause de la lenteur
  • ❌ Problèmes de compatibilité avec les serveurs HTTP non-sécurisés

Aperçu de l'interface

Note : Je n'ai pu retrouver de capture d'écran de l'application sous cette ancienne version, j'ai donc reproduit l'interface avec une maquette Figma.

Echange de données

Dans cette version, l'outil web faisait office d'intermédiaire dans l'échange de données entre l'utilisateur et le NAS, ce qui ralentissait considérablement les taux de transfert.

De plus, l'API File System Access nécessite un contexte sécurisé (HTTPS) pour fonctionner, ce qui limitait son utilisation dans l'environnement de développement et de test de Yooz qui utilisait encore des connexions HTTP non-sécurisées.
